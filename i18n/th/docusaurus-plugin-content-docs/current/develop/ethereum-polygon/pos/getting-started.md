โปรดตรวจสอบ[เอกสาร Matic.js ล่าสุดเกี่ยวกับ PoS](../matic-js/get-started.md) เพื่อเริ่มต้น

บริดจ์คือชุดสัญญาที่ช่วยในการย้ายสินทรัพย์จากเชนต้นทางไปยังเชนย่อยมีสองบริดจ์หลักสำหรับย้ายสินทรัพย์ระหว่าง Ethereum และ Polygonอันแรกคือสะพาน Plasma และอันที่สองเรียกว่า **สะพาน PoS** หรือ **Proof of Stainage Stainage Stake****บริดจ์ Plasma** ให้การรับประกันความปลอดภัยที่เพิ่มขึ้นเนื่องจากกลไกการออก Plasma

อย่างไรก็ตาม มีข้อจำกัดบางประการเกี่ยวกับโทเค็นย่อย และมีระยะเวลาการถอน 7 วันที่เกี่ยวข้องกับการออก/การถอนออกจาก Polygon ไปยัง Ethereum บนบริดจ์ Plasma

ซึ่งค่อนข้างเจ็บปวดสำหรับผู้ใช้ DApps / ผู้ที่ต้องการ**การถอน****ความยืดหยุ่น**และเร็วกว่า และมีความสุขกับระดับความปลอดภัยที่กำหนด โดยสะพาน Polygon Proof of Sake ที่ได้รับการรับรองโดยชุดตัวตรวจสอบความถูกต้องภายนอกที่แข็งแกร่ง

สินทรัพย์ที่ใช้ Proof of Stake ให้การรักษาความปลอดภัย PoS และการออกที่เร็วขึ้นด้วยช่วงเช็คพอยต์หนึ่งช่วง

## ขั้นตอนการใช้บริดจ์ PoS {#steps-to-use-the-pos-bridge}

ก่อนที่เราจะเข้าส่วนนี้ของเอกสาร มันอาจช่วยในการทำความเข้าใจเงื่อนไขอย่างละเอียดเนื่องจากคุณจะโต้ตอบกับพวกเขา ในขณะที่พยายามใช้สะพาน: [Map](https://docs.polygon.technology/docs/develop/ethereum-polygon/submit-mapping-request/) และ[กลไกSync State](https://docs.polygon.technology/docs/pos/state-sync/state-sync/)

จากนั้นขั้นตอนแรกที่ใช้สะพาน PoS คือการแผนที่ **Root Token** และ **Child Token**หมายความว่าสัญญาโทเค็นบนเชนของระบบ root และสัญญาโทเค็นบนเชนของเด็กต้องรักษาการเชื่อมต่อ (ที่เรียกว่า Mapping) เพื่อถ่ายโอนสินทรัพย์ระหว่างตัวเองหากคุณสนใจที่จะส่งคำขอการเม้าส์ โปรดใช้[คู่มือนี้](/docs/develop/ethereum-polygon/submit-mapping-request/)

ในระดับต่ำกว่าและรายละเอียดเพิ่มเติมนี่คือสิ่งที่เกิดขึ้น:

### deposit {#deposit}

  1. เจ้าของโทเค็นสินทรัพย์ **(ERC20/ERC721/ERC1155)** ต้องอนุมัติสัญญาเฉพาะบนบริดจ์ PoS เพื่อใช้จ่ายจำนวนโทเค็นที่จะโอนสัญญาเฉพาะนี้เรียกว่า **สัญญา Predicate** (ที่ปรับใช้ในเครือข่าย Ethereum) ซึ่งจริงๆ แล้ว **จะล็อกจำนวนโทเค็นที่จะฝากเอาไว้**
  2. เมื่อได้รับการอนุมัติ ขั้นตอนต่อไปคือการ**ฝากสินทรัพย์**ต้องมีการเรียกฟังก์ชัน ทำบน`RootChainManager`สัญญาซึ่งเพื่อสร้าง`ChildChainManager`สัญญาบนเชน Polygon
  3. สิ่งนี้เกิดขึ้นผ่านกลไกการซิงค์สถานะ ซึ่งสามารถทำความเข้าใจอย่างรายละเอียดได้จาก[ที่นี่](/docs/pos/state-sync/state-sync/)
  4. โดย`ChildChainManager`ภายใน จะเรียก`deposit`ฟังก์ชั่นของสัญญาโทเค็นของเด็ก และจำนวนโทเค็นที่เกี่ยวข้องจะถูก**ส่งไปยังบัญชีผู้ใช้แล้ว**เป็นสิ่งสำคัญที่จะต้องทราบว่ามีเพียงเท่านั้นที่`ChildChainManager`สามารถเข้าถึง`deposit`ฟังก์ชั่นบนสัญญาโทเค็นของเด็ก
  5. เมื่อผู้ใช้ได้รับโทเค็น **จะสามารถโอนโทเค็นแทบจะในทันที โดยมีค่าธรรมเนียมเล็กน้อยในเชน Polygon**

### การถอน {#withdrawals}

  1. การถอนสินทรัพย์กลับ Ethereum คือกระบวนการ 2 ขั้นตอน ซึ่งจะต้อง**ถูกเผาบนเชน Polygon ก่อน**แล้ว**จะต้องส่งการเผาครั้ง**นี้บนเชน Ethereum
  2. ธุรกรรมการเบิร์นใช้เวลาประมาณ 20 นาทีถึง 3 ชั่วโมงในการผ่านเช็คพอยต์ในเชน Ethereumซึ่งทำโดยผู้ตรวจสอบแบบ Proof of Stake
  3. เมื่อมีการเพิ่มธุรกรรมลงในเช็คพอยต์ จึงสามารถส่งหลักฐานของธุรกรรมการเผาบน`RootChainManager`สัญญาบน Ethereum ได้โดยเรียก`exit`ฟังก์ชัน
  4. การเรียกฟังก์ชันนี้ **จะตรวจสอบการรวมเช็คพอยต์** จากนั้นทริกเกอร์สัญญา Predicate ซึ่งล็อกโทเค็นสินทรัพย์ไว้เมื่อมีการฝากสินทรัพย์ในตอนแรก
  5. ตามขั้นตอนสุดท้าย **สัญญาที่คาดการณ์จะปล่อยโทเค็นที่ถูกล็อค**และคืนเงินให้ไปยังบัญชีผู้ใช้บน Ethereum

:::tip

เมื่อทำการแมปเสร็จแล้ว คุณสามารถใช้ **matic.js SDK** เพื่อโต้ตอบกับสัญญาหรือทำเช่นเดียวกันโดยไม่มี SDKอย่างไรก็ตาม matic.js SDK ได้รับการออกแบบมาให้เป็นมิตรกับผู้ใช้ เพื่อทำให้กลไกการโอนสินทรัพย์ง่ายต่อการรวมเข้ากับแอปพลิเคชันใดๆ

:::
